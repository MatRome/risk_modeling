Modelowanie ryzyka kredytowego: PD, LGD, EAD i Expected Loss¶

Cel projektu¶

Celem projektu jest zbudowanie kompletnego pipeline'u do oceny ryzyka kredytowego dla portfela pożyczek.

Projekt obejmuje:

  1. eksplorację i przygotowanie danych,
  2. definicję defaultu,
  3. modelowanie PD (Probability of Default),
  4. kalibrację prawdopodobieństw PD,
  5. modelowanie LGD (Loss Given Default),
  6. przyjęcie i uzasadnienie EAD (Exposure at Default),
  7. wyznaczenie Expected Loss,
  8. analizę portfela,
  9. symulację polityki odrzucania najbardziej ryzykownych kredytów,
  10. wnioski biznesowe.

2. Wstępna eksploracja danych¶

Sprawdzam strukturę danych, typy zmiennych, braki danych oraz podstawowe statystyki.

W tym kroku:

  • zamieniam procenty na wartości liczbowe,
  • zamieniam okres kredytu na liczbę miesięcy,
  • tworzę długość historii kredytowej,
  • tworzę wskaźniki relatywne, np. rata do dochodu i kwota kredytu do dochodu.

Takie zmienne często są bardziej informacyjne niż same wartości nominalne.

def oczysc_kolumne_procentowa(series):
    """
    Funkcja usuwa znak procentu z kolumny tekstowej i zamienia wartości na liczby.
    Dzięki temu zmienne takie jak oprocentowanie lub wykorzystanie limitu mogą być
    użyte w analizie i modelowaniu.
    """
    return (
        series.astype(str)
        .str.replace("%", "", regex=False)
        .str.replace("nan", "", regex=False)
        .replace("", np.nan)
        .astype(float)
    )


def oczysc_okres_kredytu(series):
    """
    Funkcja wyciąga liczbę miesięcy z kolumny opisującej okres kredytu.
    Przykładowo wartość '36 months' zostaje zamieniona na 36.
    """
    return (
        series.astype(str)
        .str.extract(r"(\d+)")[0]
        .astype(float)
    )


def oblicz_miesiace_od_daty(series):
    """
    Funkcja zamienia datę pierwszej linii kredytowej na liczbę miesięcy historii kredytowej.
    Jako punkt odniesienia przyjmuje najnowszą datę występującą w kolumnie.
    """
    dates = pd.to_datetime(series, errors="coerce")
    data_odniesienia = dates.max()

    miesiace = (data_odniesienia.year - dates.dt.year) * 12 + (
        data_odniesienia.month - dates.dt.month
    )

    return miesiace

3. Definicja defaultu¶

W modelowaniu ryzyka kredytowego najpierw trzeba zdefiniować, które kredyty uznajemy za default.

W danych Lending Club typowe statusy defaultowe to m.in.:

  • Charged Off,
  • Default,

Za dobre kredyty przyjmuję statusy zakończone pełną spłatą, np. Fully Paid.

Kredyty aktywne lub niejednoznaczne usuwam z modelowania, ponieważ nie znamy jeszcze ich finalnego wyniku.

def ocen_model(nazwa_modelu, model, X_test, y_test):
    """
    Funkcja ocenia model klasyfikacyjny.
    Wyznacza predykcję klasy, prawdopodobieństwo defaultu oraz podstawowe miary jakości.
    """
    y_pred = model.predict(X_test)
    y_prob = model.predict_proba(X_test)[:, 1]

    wyniki = {
        "model": nazwa_modelu,
        "auc": roc_auc_score(y_test, y_prob),
        "gini": 2 * roc_auc_score(y_test, y_prob) - 1,
        "accuracy": accuracy_score(y_test, y_pred),
        "precision": precision_score(y_test, y_pred),
        "recall": recall_score(y_test, y_pred),
        "f1": f1_score(y_test, y_pred)
    }

    print("=" * 60)
    print(nazwa_modelu)
    print("=" * 60)
    print("AUC:", round(wyniki["auc"], 4))
    print("Gini:", round(wyniki["gini"], 4))

    print("\nMacierz pomyłek:")
    print(confusion_matrix(y_test, y_pred))

    print("\nRaport klasyfikacji:")
    print(classification_report(y_test, y_pred))

    return wyniki

def pobierz_nazwy_zmiennych_z_preprocessingu(preprocessing, zmienne_numeryczne, zmienne_kategoryczne):
    """
    Funkcja pobiera nazwy zmiennych po preprocessingu.
    Dla zmiennych numerycznych zostawia oryginalne nazwy, a dla zmiennych kategorycznych
    pobiera nazwy utworzone po kodowaniu one-hot.
    """
    nazwy_zmiennych = []

    nazwy_zmiennych.extend(zmienne_numeryczne)

    if len(zmienne_kategoryczne) > 0:
        pipeline_kategoryczny = preprocessing.named_transformers_["kategoryczne"]
        kodowanie = pipeline_kategoryczny.named_steps["kodowanie"]

        nazwy_kategoryczne = kodowanie.get_feature_names_out(
            zmienne_kategoryczne
        ).tolist()

        nazwy_zmiennych.extend(nazwy_kategoryczne)

    return nazwy_zmiennych

Wnioski końcowe¶

  1. Model PD osiąga umiarkowaną zdolność dyskryminacyjną (AUC ~0.72), co oznacza, że poprawnie rozróżnia klientów lepszych od gorszych, ale pozostawia przestrzeń do dalszej optymalizacji.

  2. Model ma tendencję do zawyżania prawdopodobieństwa defaultu, co potwierdza analiza kalibracji – może to być celowe (konserwatywne podejście do ryzyka), ale wymaga ewentualnej kalibracji.

  3. LGD jest bardzo wysokie (średnio ~93%), co oznacza, że w przypadku defaultu bank odzyskuje niewielką część należności – portfel jest silnie narażony na straty.

  4. Model LGD ma niską zdolność wyjaśniania zmienności (niskie R²), co sugeruje, że strata po default jest trudna do przewidzenia na podstawie dostępnych zmiennych.

  5. Największy wpływ na ryzyko mają zmienne finansowe klienta (oprocentowanie, DTI, wykorzystanie limitów), co jest spójne z praktyką kredytową.

  6. Analiza segmentów pokazuje, że ryzyko rośnie wraz z pogorszeniem ratingu oraz jest wyższe dla niektórych celów kredytowych (np. small_business), co umożliwia segmentację portfela.

  7. Symulacja strategii kredytowej pokazuje, że odrzucenie najbardziej ryzykownych klientów znacząco obniża poziom Expected Loss, co potwierdza praktyczną użyteczność modelu w zarządzaniu portfelem.
